# codex-notes 跨电脑初始化指南

## 1. 目标

在一台新电脑上完成以下初始化：
- 连接 GitHub（SSH）
- 拉取并使用 `codex-notes`
- 恢复 `save_md` 能力（先 pull、再归档、提交、按需 push）

## 2. 前置条件

- 系统：macOS
- 已安装：`git`、`zsh`、`ssh`
- 具备 GitHub 账号权限（`Jerry7379/codex-notes`）

可选安装（推荐）：

```bash
brew install ripgrep
```

## 3. Git 与 SSH 初始化

### 3.1 配置 Git 身份

```bash
git config --global user.name "Jerry7379"
git config --global user.email "nierui@huoban.ai"
```

### 3.2 生成 GitHub 专用 SSH key

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
ssh-keygen -t ed25519 -C "nierui@huoban.ai" -f ~/.ssh/id_ed25519_github -N ""
```

### 3.3 写入 SSH 配置

```bash
cat >> ~/.ssh/config <<'EOF_CONFIG'

Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_github
  AddKeysToAgent yes
  UseKeychain yes
  IdentitiesOnly yes
EOF_CONFIG

chmod 600 ~/.ssh/config
ssh-add --apple-use-keychain ~/.ssh/id_ed25519_github
```

### 3.4 添加公钥到 GitHub

```bash
cat ~/.ssh/id_ed25519_github.pub
```

把输出内容粘贴到：
`https://github.com/settings/keys`

验证连接：

```bash
ssh -T git@github.com
```

预期输出包含：`Hi Jerry7379! You've successfully authenticated`

## 4. 拉取 codex-notes

```bash
git clone git@github.com:Jerry7379/codex-notes.git ~/codex-notes
cd ~/codex-notes
git checkout main
git pull --rebase
```

## 5. 恢复 save_md 技能

当前 `save_md` 位于旧电脑路径：`~/.codex/skills/save_md`。
新电脑有两种恢复方式：

### 方案 A（推荐）：从旧电脑复制技能目录

```bash
mkdir -p ~/.codex/skills
# 把 old-mac-host 替换成旧电脑可访问地址
rsync -avz old-mac-host:/Users/nrick/.codex/skills/save_md ~/.codex/skills/
```

### 方案 B：手工创建（无旧电脑连接时）

手工创建：
- `~/.codex/skills/save_md/SKILL.md`
- `~/.codex/skills/save_md/agents/openai.yaml`
- `~/.codex/skills/save_md/scripts/archive_doc.sh`

然后赋权：

```bash
chmod +x ~/.codex/skills/save_md/scripts/archive_doc.sh
```

## 6. 首次演练（验收）

先准备一篇测试文档：

```bash
cat > /tmp/test-save-md.md <<'EOF_TEST'
# save_md 初始化验收

用于验证跨电脑初始化后流程可用。
EOF_TEST
```

执行归档（不 push）：

```bash
bash ~/.codex/skills/save_md/scripts/archive_doc.sh \
  /tmp/test-save-md.md \
  --repo ~/codex-notes \
  --keywords "初始化,跨电脑,save_md"
```

验收检查：

```bash
git -C ~/codex-notes status --short --branch
git -C ~/codex-notes log --oneline -n 1
rg -n "save_md 初始化验收|跨电脑" ~/codex-notes/INDEX.md
```

若需要推送远端：

```bash
git -C ~/codex-notes push
```

## 7. 常见问题

### 7.1 `Permission denied (publickey)`

说明 SSH 公钥未加到 GitHub，或 `~/.ssh/config` 未指向 `id_ed25519_github`。

### 7.2 `没有可提交的变更`

说明文档和索引没有变化，属于正常提示。

### 7.3 `main...origin/main [ahead N]`

说明本地有提交未推送，执行：

```bash
git -C ~/codex-notes push
```

## 8. 日常使用建议

- 归档默认不推送，先看提交内容再决定是否 `push`。
- 每次换机后优先跑第 6 节验收命令，确认整套流程已恢复。
